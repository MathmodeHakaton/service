# Задачи пайплайна

## Задача 1. Собрать daily dataset
- Привести даты.
- Размазать месячные/недельные/event-данные на дни.
- Не заполнять длинные разрывы вроде пятилетней дырки OFZ.
- Считать `full_model_valid`.

## Задача 2. Посчитать rolling MAD
- Baseline считается только по прошлому окну.
- Текущий день не входит в свою норму.
- Для долгих рядов использовать окно около 3 лет.

## Задача 3. Сделать stress components
- Не считать ручной module score через веса.
- Каждый показатель становится отдельной компонентой.
- Для каждого показателя задается направление стресса.

## Задача 4. Сделать weak target
- Использовать state rules: active modules, сильные комбинации M2+M5 / M2+M3 / M2+M3+M5.
- Делать target непрерывным, не круглыми классами.
- M4 использовать как контекст сезонности.
- Для early-warning подтягивать target к будущему стрессу в горизонте H дней.

## Задача 5. Обучить CatBoost
- Вход: stress components + context + coherence + interactions.
- Выход: LSI 0-100.
- Финальный LSI = prediction CatBoost, не ручная сумма.

## Задача 6. Объяснить через SHAP
- Посчитать SHAP по признакам.
- Сгруппировать SHAP по модулям.
- Показывать dynamic contribution на дашборде.

## Задача 7. Валидация
- Backtest по доступным кризисным окнам.
- 2014 опустить, если нет полного OFZ-покрытия.
- Для честной проверки делать leave-one-crisis-out.
