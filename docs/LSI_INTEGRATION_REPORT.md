2. **Страница LSI** (`src/presentation/pages/6_LSI.py`) полностью переписана:

     с расшифровкой: lsi_smoothed, lsi_raw, множитель M4.
   - Горизонтальный bar-chart SHAP-вкладов M1..M5 на сегодня (зелёный =
     тянет вниз, красный = тянет вверх). Это и есть «вклад модулей»,

   - Раскрывающиеся блоки: глобальная важность модулей, top-15 фич,
     backtest по стресс-эпизодам.
   - Таймсерия LSI 2016→сегодня с подсветкой статуса по дням, порогами
     40/70 и вертикальными зонами кризисов из ТЗ.
   - Блок LLM-комментария (см. ниже).

3. **LLM-интеграция** — новый модуль `src/presentation/llm_commentary.py`:
   - Промпт строится из шаблона ТЗ («БОНУС LLM-модуль») с подстановкой
     LSI, вкладов модулей, активных флагов и ближайших налоговых дат.
   - Поход в **Ollama** (HTTP `/api/generate`). Модель по умолчанию —
     `qwen2.5:3b-instruct` (Q4_K_M, ~2 ГБ). Поднимается одной командой:
     ```
     ollama serve
     ollama pull qwen2.5:3b-instruct
     ```
     При желании — `qwen2.5:7b-instruct` (выше качество, медленнее).
   - Ответ кешируется на 10 минут (`st.cache_data`), чтобы не дёргать
     модель при каждом ререндере.
   - Если Ollama недоступна или модель не установлена — страница
     выводит понятную инструкцию по установке + детерминированный
     текстовый фолбэк, собранный из вкладов SHAP и флагов. Дашборд
     остаётся рабочим без LLM.

4. **Бэкенд не тронут.** Изменения только в `presentation/`. `application/`,
   `domain/`, `infrastructure/` остались как есть. Существующий

## Почему такие решения

- **Чтение CSV вместо реального инференса CatBoost.** ТЗ требует пайплайн,
  но обученная модель уже посчитала LSI на весь исторический горизонт
  (2016 — май 2026). Грузить её повторно при каждом запуске Streamlit
  бессмысленно — CSV даёт мгновенный отклик. Когда понадобится свежий
  день — рядом лежит `.cbm` и метаданные, можно дописать тонкий wrapper
  без изменений в UI.
- **SHAP-вклады модулей в bar-chart.** Это прямое выполнение
  обязательного требования из раздела «LSI Агрегационный слой» ТЗ:
  *«для каждого значения LSI необходимо показать вклад каждого модуля в
  итоговый сигнал»*. Вклад M4 показывается как мультипликативный эффект
  (lsi_raw − lsi_base), что соответствует архитектуре модели.
- **Ollama + квантованный Qwen, а не llama-cpp-python / HF.** В проекте
  уже подключен пакет `ollama` и есть `LocalLLM`-обёртка — значит,
  Ollama-демон уже задуман как способ доставки моделей. Для Windows это
  самый прямой путь: один установщик, авто-квантизация, HTTP API. Qwen
  3B Q4 — оптимум по «качество / память / скорость на CPU» для коротких
  3–5 предложений на русском.
- **Промпт по ТЗ.** Шаблон взят из задания (раздел «Пример структуры
  промпта»), дополнен явной структурой ответа из вашего сообщения
  (что сейчас / какие модули давят / чего ждать). `temperature=0.3` —
  чтобы не уходило в воду на финансовых формулировках.

## Как проверить

```powershell
# 1) поднять LLM (один раз)
ollama serve            # в отдельном окне
ollama pull qwen2.5:3b-instruct

# 2) запустить дашборд
streamlit run src/presentation/app.py --server.port 8501
```

Откройте боковое меню → «LSI». Сверху — гейдж и статус, ниже —
SHAP-вклады, история LSI, backtest, в самом низу — LLM-комментарий.
